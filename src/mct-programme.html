Programme overview

The July-October schedule is framed as a single learning journey, not six disconnected events. Each weekend adds a layer: first the platform, then operations, design, governance, and applied AI.

July: foundation

Start with Azure Fundamentals and Azure AI Fundamentals to establish the shared vocabulary.

August: platform skills

Move into Azure Administrator topics and the operational practices needed to run cloud workloads.

September: design and control

Build architecture judgment, then connect it to security and governance decisions.

October: AI engineering

Finish with Azure AI Engineer skills that turn the foundation into applied AI delivery capability.

Weekend course sequence

The courses below follow the source order from the 2026 course material and are presented as one coherent July-October progression.

Course 1

Azure Fundamentals

Begin with the core Azure concepts, services, pricing, support, and cloud vocabulary needed for the full path.

  • Path role Establishes the cloud baseline for every later weekend session.
Course 2

Azure AI Fundamentals

Connect cloud basics to AI workloads, responsible AI concepts, and the services used to build intelligent solutions.

  • Path role Gives learners the AI foundation needed before deeper engineering work.
Course 3

Azure Administrator

Build the operational skills for identity, compute, storage, networking, monitoring, and resource management.

  • Path role Turns the foundation into hands-on platform administration capability.
Course 4

Azure Architecture

Move from operating services to designing reliable, scalable, maintainable cloud solutions.

  • Path role Develops architecture thinking for workload design and technical trade-offs.
Course 5

Azure Security / Governance

Add the controls, policies, identity practices, and governance patterns needed for responsible cloud adoption.

  • Path role Connects architecture decisions to risk, compliance, and operational control.
Course 6

Azure AI Engineer

Apply the programme foundation to AI solution design, integration, evaluation, and delivery practices.

  • Path role Completes the journey with applied AI engineering readiness.
